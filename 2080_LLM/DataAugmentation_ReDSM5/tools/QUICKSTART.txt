PARALLEL AUGMENTATION - QUICK START
================================================================================

OPTION A: PROGRESSIVE K-VALUE GENERATION (RECOMMENDED)
-------------------------------------------------------

Single command to generate k=2, k=3, k=4 with decision gates:
   $ ./tools/progressive_k_runner.sh

Or auto-approve all phases:
   $ ./tools/progressive_k_runner.sh --auto-approve

Generate larger k values (k=2 to k=28):
   $ ./tools/progressive_k_runner.sh --start-k 2 --end-k 28 --auto-approve


OPTION B: MANUAL K-VALUE GENERATION
------------------------------------

1. Launch augmentation for specific k value (e.g., k=2):
   $ MAX_COMBO_SIZE=2 ./tools/run_parallel_augment.sh

2. Monitor progress (in another terminal):
   $ python tools/monitor_augment.py --follow

3. Merge results:
   $ ./tools/merge_shard_manifests.sh
